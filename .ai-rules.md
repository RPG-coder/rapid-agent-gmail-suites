# 🤖 Gemini-CLI Workspace Rulebook: Rapid Agent Gmail Suite

This workspace contains a specialized, event-driven multi-agent Google Workspace Add-on built for the Google Rapid Agent Hackathon. All AI generations must conform to these architectural guardrails.

---

## 📐 1. Global Architectural Constraints

- **Language:** Python 3.11+ for all agent business logic and data streaming.
- **Framework:** Google Agent Development Kit (ADK) paired with standard Model Context Protocol (MCP) using `fastmcp`.
- **UI Delivery:** Google Workspace Card Service (Gmail Sidebar UI via Apps Script in `publish/sidebar-ui/`).
- **Deployment Paradigm:** Serverless containers (Google Cloud Run) orchestrated via Pub/Sub and EventArc triggers.
- **Blueprint Pattern:** Use `agents/smart-email-manager/` as the structural reference for all new agents.

---

## 📂 2. Directory Boundaries & Code Ownership

You must isolate file generation to their respective boundaries. Never mix domain logic. All agents (`inbox-analytics`, `self-arizeing-manager`) must mirror the `smart-email-manager` structure:
- `main.py`: FastAPI entry point and MCP discovery. Maps HTTP/PubSub triggers to toolbox orchestrators.
- `toolbox.py`: High-level business logic and workflow orchestrators (e.g., `reorganize_mails`, `perform_batch_classification`).
- `tools/`: Low-level, atomic MCP tool definitions (e.g., `gmail_mcp.py`, `mongo_mcp.py`).
- `db/` & `config/`: Localized client initialization (e.g., `mongo_client.py`) and settings management.

### 🔹 `agents/smart_email_manager/` (Blueprints)
- **Responsibility:** Real-time email categorization and semantic grouping.
- **Implementation Example:** Look at `toolbox.py` for how it combines `gmail_mcp` and `mongo_mcp` tools.

### 🔹 `agents/inbox_analytics/`
- **Responsibility:** Analytical RAG, computing email velocity patterns, and response metrics.
- **Implementation Standard:** Must mirror SEM's 3-tier structure. Use BigQuery for analytical queries.

### 🔹 `agents/self_arizeing_manager/`
- **Responsibility:** LLM observability (Arize Phoenix) and automated prompt performance tuning.
- **Implementation Standard:** Must mirror SEM's 3-tier structure. Uses Gemini 3.5 as a "Judge" to evaluate traces.

---

## 🛠️ 3. Execution & Code Generation Guardrails

### A. 3-Tier Logic Architecture (Mandatory)
1. **API Layer (`main.py`):** REST/PubSub endpoints. Calls `toolbox.py`.
2. **Orchestration Layer (`toolbox.py`):** "The Brain". Performs multi-step operations using Tools.
3. **Tool Layer (`tools/`):** "The Hands". Functions decorated with `@mcp.tool()`.

### B. Model Context Protocol (MCP) Compliance
- Always use `FastMCP` for tool definition.
- Wrap all MCP network and tool communication within asynchronous (`async/await`) handlers where possible.

### C. Invisible Telemetry
- Business logic must remain clean.
- All tracing and observability hooks must be abstracted in `agents/self_arizeing_manager/telemetry.py`.

### D. AI Model Selection (Hackathon Requirement)
- **Primary Model:** `gemini-3.5-flash` for high-speed classification, naming, and Judge tasks.
- **Reasoning Model:** `gemini-3.5-pro` for complex analytical reasoning in `inbox_analytics`.

---

## 🎯 4. Code Generation Workflows (Prompt Shortcuts)

- **Task Type: Scaffold New Agent**
  - _Action:_ Initialize the directory structure mirroring SEM (main, toolbox, tools/, db/, config/).
- **Task Type: Implement MCP Tool**
  - _Action:_ Define the tool in `tools/[domain]_mcp.py`, then register it in `main.py`'s `TOOLS` dictionary.
- **Task Type: Formulate UI Card Handshake**
  - _Action:_ Keep backend logic execution lightweight. Apps Script is stateless; pass state in `Action.setParameters()`.
